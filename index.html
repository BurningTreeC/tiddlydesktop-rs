Privacy Policy for TiddlyDesktop-RS

Last updated: February 25, 2026

Introduction

TiddlyDesktop-RS ("we", "our", or "the app") is an open-source application for viewing and editing TiddlyWiki files. We are committed to protecting your privacy. This Privacy Policy explains how we handle your data.

Data Collection

We do not collect any personal data. No data is ever transmitted to any server unless you explicitly opt in to the optional sync features described below.

TiddlyDesktop-RS is designed to work locally on your device. The app:

  • Does not collect personal information
  • Does not track your usage
  • Does not use analytics services
  • Does not use advertising
  • Does not transmit any data to external servers unless you explicitly enable optional sync features

Local Data Storage

TiddlyDesktop-RS stores the following data locally on your device:

  • Wiki Files: Your TiddlyWiki files are stored in locations you choose on your device's storage.
  • Recent Wikis List: A list of recently opened wikis is stored locally to provide quick access.
  • Favicon Cache: Extracted favicon images from your wikis are cached locally for display in the app.
  • Backup Files: If backups are enabled, copies of your wiki files are stored locally.
  • Sync Configuration: If you enable sync features, connection settings (device name, room codes, encrypted passwords, and encrypted authentication tokens) are stored locally on your device.

All locally stored data remains on your device and is never transmitted elsewhere unless you explicitly enable sync features.

Optional Sync Features (Opt-In Only)

The following features are entirely optional and disabled by default. No data leaves your device unless you explicitly choose to enable them.

LAN Sync

When you enable LAN Sync, the app discovers and communicates with other TiddlyDesktop-RS devices on your local network to synchronize wiki content. All communication stays within your local network and does not pass through any external servers. Wiki content is encrypted end-to-end between devices.

Relay Sync

When you enable Relay Sync, the app can synchronize wiki content between devices over the internet via a relay server. This feature requires you to log in with a GitHub account and involves the following:

  • GitHub Authentication: When you choose to log in with GitHub, the app initiates a standard OAuth authorization flow. Your GitHub username and user ID are shared with the relay server to verify your identity and manage room membership. We only request the read:user scope and do not access your repositories, email, or any other GitHub data beyond your public username and ID.
  • Relay Server: The relay server (relay.tiddlydesktop-rs.com) facilitates message passing between your devices. Room membership data (GitHub usernames associated with rooms) is stored on the relay server.
  • End-to-End Encryption: All wiki content transmitted through the relay server is end-to-end encrypted using a password-derived key that only you and your room members know. The relay server cannot read your wiki content.
  • Token Storage: Your GitHub access token is stored locally on your device, encrypted at rest using a device-specific key. It is never stored in plaintext.

Network Access

The app uses network access for:

  • Running a local web server for folder-based TiddlyWiki wikis (localhost only)
  • Loading local wiki content in the WebView component (localhost only)
  • Checking for app updates (a single request to GitHub to compare version numbers)
  • LAN Sync device discovery and communication (local network only, opt-in)
  • Relay Sync communication with the relay server (internet, opt-in)
  • GitHub OAuth authentication (internet, opt-in)

Third-Party Services

TiddlyDesktop-RS does not use any analytics platforms or advertising networks. The following third-party services are used only when you explicitly opt in:

  • GitHub OAuth: Used for authentication with the relay server. Subject to GitHub's Privacy Statement.
  • Relay Server: Operated by the TiddlyDesktop-RS project. Stores room membership data only. Cannot access your wiki content (end-to-end encrypted).

Data Retention

  • Local data: Stored until you delete it or uninstall the app.
  • Relay server data: Room membership records are stored until a room owner deletes the room. GitHub authentication tokens are cached on the relay server for up to 5 minutes for validation purposes.

Children's Privacy

Our app does not knowingly collect any personal information from anyone, including children under the age of 13.

Open Source
